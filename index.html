📈 Stock model
Weekly schedule
Monday morning Run model, upload CSV to dashboard
Friday Log exits before weekend, review performance
Terminal command
cd ~/Library/Mobile\ Documents/com\~apple\~CloudDocs/Trading && python3 stock_model/main.py
Paste this into Terminal. Candidates CSV saves automatically.
Monday morning checklist
1Run main.py from terminal (command above)
2Find the CSV in Trading/stock_model/Results/candidates/
3Go to Stock tab → "Load Monday slate" → upload that CSV
4Review candidates, add trades you're taking via "Log a trade" form
Fidelity execution — at open
1Pre-market: open one browser tab per trade in Fidelity, stage each as a Limit Buy at the CSV entry price (Friday's close). Limit orders fill at your price or better — you never overpay. If a stock opens above your limit, the order won't fill and you skip that name cleanly.
2At 9:30: submit all limit orders. Check fills within 5–10 minutes. Cancel any unfilled orders — do not chase a name that opened above your limit price.
3After each fill: immediately place a separate Sell → Stop Loss order at the CSV stop price. Set Time in Force to GTC (Good Till Cancelled). Do not move to the next trade until the stop is placed.
⚠A position without a GTC stop loss is a rule violation. The stop order costs no additional capital — it just protects what you already own.
Friday checklist
1Check each open position — did target or stop trigger this week?
2Log exits for closed positions using the close button on each trade
3Archive the week via "Archive week" button before running next week's model

₿ Crypto model
Weekly schedule

Thursday night Run main.py — ETH daily signal check

Terminal command
cd ~/Library/Mobile\ Documents/com\~apple\~CloudDocs/Trading && python3 crypto_model/crypto/main.py
Paste into Terminal on Friday, Monday, and Thursday nights. Signal auto-writes to Firestore.
Friday night checklist
1Run main.py after market close (command above)
2Check all three filters: MA20, ATR expansion, momentum
3Signal auto-writes to Firestore — open Crypto tab, it's already there
4Log LONG or NO_TRADE in dashboard — one row per instrument
5If LONG: place stop-limit order in TradingView paper account
6Export JSON backup → save to Trading/crypto_model/
Sunday night checklist
1Check TradingView — did target or stop trigger?
2If neither: TIME exit at current market price ~9pm ET
3Log exit in dashboard — exit price, exit type, notes
4Close week in Weekly tab with a short note
5Export JSON backup → save to Trading/crypto_model/ with today's date
Exit types explained
TARGET2R level hit — TradingView take-profit triggered
STOPStopped out — TradingView stop-loss triggered
TIMESunday close, neither target nor stop triggered (~78% of backtest trades)
MONDAYProfitable at Sunday close — carry with breakeven stop, close Monday
